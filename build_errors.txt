**错误原因**

这个错误是由于 `markupsafe` 库版本不兼容导致的。在 Sphinx 3.x 中，`jinja2` 库依赖于 `markupsafe` 库，而 `markupsafe` 库在版本 2.0.0 中删除了 `soft_unicode` 函数。因此，当 Sphinx 尝试导入 `soft_unicode` 时，会抛出 `ImportError`。

**修复方法**

1. **升级 `markupsafe` 库**: 可以通过 pip 升级 `markupsafe` 库到版本 2.0.0 或更高版本。运行以下命令：
```bash
pip install --upgrade markupsafe
```
2. **降级 `jinja2` 库**: 如果升级 `markupsafe` 库不行，可以尝试降级 `jinja2` 库到版本 2.x。运行以下命令：
```bash
pip install jinja2==2.11.3
```
**需要检查的文件和配置**

1. **`requirements.txt` 文件**: 检查项目的 `requirements.txt` 文件是否指定了 `markupsafe` 和 `jinja2` 的版本。如果指定了版本，请确保它们是兼容的。
2. **`conf.py` 文件**: 检查 Sphinx 配置文件 `conf.py` 中是否有任何关于 `jinja2` 或 `markupsafe` 的配置。

**预防措施**

1. **保持依赖库更新**: 定期检查和更新项目中的依赖库，以避免版本不兼容的问题。
2. **使用虚拟环境**: 使用虚拟环境（如 conda、virtualenv 等）可以隔离项目的依赖库，避免全局安装导致的冲突问题。
3. **指定依赖库版本**: 在 `requirements.txt` 文件中指定依赖库的版本，可以确保项目在不同环境中的兼容性。
